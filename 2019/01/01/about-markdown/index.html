关于markdown的一些使用

###

  1. 这几天在看blog或论文就会发现，看的越多，到实际下手的时候发现无从下手，
    究其原因还是当时看的多，思考的少，没有记录下来整个过程。
    因此要把被动学习变成主动学习，多写写，即时踩到坑，以后翻出来看看也是一种
    生活过程。废话不多说，走起。
  2. 首先就是hexo支持markdown，通常我们写的Blog都是通过md文件解析为静态网页的。
    还有就是最近在看github上的技术文档时，也发现了一般都是用md文件。了解md文件的
    格式尤为重要。
  3. Markdown 是一种用来写作的轻量级「标记语言」，它用简洁的语法代替排版，而不像一般我们用的字处理软件 Word 或 Pages 有大量的排版、字体设置。它使我们专心于码字，用「标记」语法，来代替常见的排版格式。
    例如此文从内容到格式，甚至插图，键盘就可以通通搞定了。
  4. 链接引用格式 链接的文字放在[]中，链接地址放在随后的（）中，举一反三
    eg.[文章引用](https://www.cnblogs.com/liugang-vip/p/6337580.html)
    插入链接与插入图片的语法很像，区别在一个 !号

  5. 关于代码框 用来展示代码 如果代码量比较少，只有单行的话，可以用单反引号包起来，如下：
    <p><a href="www.baidu.com"></a></p>
    要是多行这个就不行了，多行可以用这个：

    <p><a href="www.baidu.com"></a></p> <p><a href="www.baidu.com"></a></p> <p><a href="www.baidu.com"></a></p>

  6. 如果一段文字被定义为标题，只要在这段文字前加 # 号即可。
    eg # 一级标题 ## 二级标题 以此类推，总共六级标题，建议在井号后加一个空格，这是最标准的 Markdown 语法。

  7. 粗体与斜体
    用两个 包含一段文本就是粗体的语法，用一个 包含一段文本就是斜体的语法。

  8. 关于文章资源引用

  9. 分割线
    一行连用三个或者三个以上的星号,减号,或者下划线,就可以表示分割线.
